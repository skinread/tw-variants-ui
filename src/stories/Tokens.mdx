# Design Tokens

These tokens reflect what is configured in code and are a _work in progress_,
they do not yet reflect all design decisions nor the final naming convention.

### Theme overview

Themes should be comprised of semantic colours, that is a colour labelled with a purposeful name rather than the colour itself.
There are many different categorisations and naming conventions for semantic colours based on what the brand(s) require, colour modes, etc.

Alternatively, palette colours are static values with a label that references the shade itself, such as _Blue 600_ or `blue.600` and would always be the same regardless of the theme.

A few semantic naming pattern examples include:

- **interaction** main / focus / hover / pressed
- **inverse** background / foreground
- **neutral** 100 / 200 / 300 / etc
